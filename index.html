    Logic Explanation

    All loot is kept in one array called lootItems. Each time the Add Loot button is clicked, the program builds a small object that holds the item's name and value and pushes that object onto the array, so the data keeps growing across clicks instead of resetting each time.

    Loops are used in two places. One for loop adds up every item's value to produce the Total Loot, and a second for loop walks through the array to rebuild the on-screen loot list each time it changes. The same total loop is reused when splitting the loot.

    Conditional logic (if statements) runs inside the validation functions before any data is stored or split. It blocks empty names, missing or non-numeric values, and negative values from entering the array. Before a split, it also checks that at least one loot item exists and that the party size is one or greater. When a check fails, a message is shown inside the page instead of an alert.
